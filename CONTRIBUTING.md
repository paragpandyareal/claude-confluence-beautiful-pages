# Contributing to Claude Confluence Beautiful Pages

Thanks for your interest in improving this skill! Contributions are welcome and appreciated.

## How to Contribute

1. **Fork** this repository to your own GitHub account
2. **Clone** your fork locally
3. **Create a branch** for your changes (e.g. `fix/status-lozenge-localid` or `feature/expand-layout`)
4. **Make your changes**
5. **Test your changes** by using the skill in Claude Code or Claude.ai to create a Confluence page
6. **Submit a Pull Request** back to this repository

## What Makes a Good Contribution

- New ADF building blocks or patterns that others would find useful
- Additional page templates for common use cases
- New gotchas you've discovered through real usage
- Bug fixes (e.g. ADF snippets that don't render correctly)
- Improvements to the documentation or README
- Performance tips (e.g. better ways to stay under the 30KB limit)

## Pull Request Guidelines

- Describe what you changed and why
- If you're adding a new gotcha, explain the problem you hit and how you discovered it
- If you're adding a template, include a brief description of the use case
- Keep changes focused. One PR per feature or fix is ideal

## What We Won't Accept

- Changes that remove the original attribution or license
- Adding personal Confluence Cloud IDs, space IDs, or other environment-specific details (the skill must stay generic)
- Changes that break existing ADF patterns without good reason
- Unrelated content or promotional material

## Code of Conduct

Be respectful and constructive. We're all here to make Confluence pages look better.

## Questions?

Open an issue if you're not sure whether a change would be welcome. Happy to discuss before you put in the work.
